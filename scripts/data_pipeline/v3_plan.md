# SSAA v3 — tool-augmented annotation, design & task plan

Source: discussion 2026-06-02 (Topic A locked, Topic B locked,
moving to tool-use rewrite).

## What's being built

Replace the "push" annotation paradigm (extractor pre-computes all
pose deltas, VLM reads everything at once) with a **pull** paradigm:
VLM decides the phase boundary first (based on semantic intent),
then queries pose data on demand via tools.

The schema also gains two fields from Topic A:
`imitation_supervised: bool` and `chunk_end_frame: int | null`.
Description field replaces episode-level `plan`; plan content moves
into kf[0]'s `A_pred`.

## Tools spec (LOCKED 2026-06-02)

```python
# tools.py — registered for the VLM subagent

def get_pose_delta(ep_path: str, idx1: int, idx2: int) -> dict:
    """
    Delta pose from frame idx1 to frame idx2.
    - delta_robot is in ROBOT BASE FRAME (forward/left/up cm).
    - delta_ee   is in WRIST CAMERA FRAME at idx1 (i.e. uses the
                 source-frame EE orientation as the reference). This
                 is the wrist-view-aligned delta.
    Also returns rotation decomposition in both frames, and any
    interaction events between idx1 and idx2.
    """
    return {
        "delta_robot": {"forward": float, "left": float, "up": float},
        "delta_ee":    {"forward": float, "left": float, "up": float},
        "delta_rot_world": str,           # e.g. "≈12° pitch+5° roll"
        "delta_rot_ee":    str,           # e.g. "≈12° pitch_ee+5° roll_ee"
        "n_frames": int,                  # idx2 - idx1
        "interaction_events_in_range": [
            {"frame_idx": int, "type": str},   # grasp/release/retry inside
        ],
        "gap_to_grasp":   {...} | None,   # to first grasp AFTER idx2
        "gap_to_release": {...} | None,   # to first release AFTER idx2
    }


def get_image(ep_path: str, frame_idx: int, view: Literal["ext", "wrist"]) -> bytes:
    """
    Return JPEG bytes (base64-ready) of the camera image at frame_idx.
    frame_idx need NOT be a keyframe — any frame in the trajectory.
    """


def get_keyframe_list(ep_path: str) -> list[dict]:
    """
    All rule-detector keyframes (from keyframe.py). Structural reference
    only, no images/deltas.
    """
    return [
        {
            "kf_idx": int,                       # 0-indexed
            "frame_idx": int,                    # video frame number
            "type": str,                         # begin/grasp/release/retry/motion/filler/end
            "gripper_state": str,                # open/partial/closed
            "near_interaction": bool,            # within ±2 of grasp/release/retry
            "interaction_context": str | None,   # pre_grasp/post_grasp/pre_release/post_release
        },
        ...
    ]
```

Decisions:
- chunk_end_frame_hint NOT included in get_keyframe_list (avoid anchoring)
- get_pose_delta's delta_ee uses idx1's EE rotation (source-frame
  visual frame, consistent with current Δee semantics)

## Design lessons to carry into v3 (from past prompt iterations)

These principles must survive the rewrite. Each emerged from a
specific failure or success in earlier rounds.

1. **Field role clarity** — each label's loss-position written at top
2. **Named patterns** — every rule gets a short name; discuss by name
3. **One concept per bullet** — never combine 2 rules
4. **Examples are last resort** — bloat-prone, prefer principles
5. **Trust the agent** — avoid hard thresholds, let VLM self-judge
6. **Causal anchor** — non-Markov; reference past actions in S
7. **Object-centric S_pred** — object relations, not arm motion
8. **Single-frame economy** — A picks robot OR wrist, not both
9. **Affordance call-out** — A mentions alignment/obstacle/contact
10. **Demo intent on retry** — A explains intent, not just numbers
11. **Failure stance** — pre-failure diverge, post-failure recovery
12. **First-frame plan** — plan in kf[0] A_pred, not separate field
13. **Selective view** — don't describe both cams every kf
14. **Avoid peek-ahead in A_pred** — reason from current obs
15. **Numbers as tool not goal** — use when precision is right level
16. **"Justify number from current obs alone" rule of thumb**
17. **Hint priority** — human hint overrides VLM perception
18. **`description` vs plan** — what happened vs how to succeed
19. **Dual id labels** — image labels show frame_idx AND kf_idx
20. **Calibrated wrist frame** — Δee from hand-eye, not empirical
